# Kaizen Rules

Permanent rules learned from mistakes.

## Default Rules

- Do not exceed 24px heading size unless task explicitly asks.
- Do not modify global design rules unless task explicitly asks.
- Do not invent UI patterns.
- Screenshot/Figma wins over model memory.
- Decisions in `makato/knowledge/decisions.md` win over summaries.
- Do not paste raw transcripts into Makato.
- Use Radius role matrix for permissions.
- Web uses light mode default.
- Mobile uses dark mode default.
- Use shadcn/ui naming for web components.
- Always note missing information as `unknown`.
- Every tool must update handoff/latest.md in JSON output.
