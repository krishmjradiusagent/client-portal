# shadcn/ui Truth

Use shadcn component naming and behavior for web work.

## Component Mapping

| Need | Component |
|---|---|
| Primary action | Button |
| Modal | Dialog |
| Side panel | Sheet |
| Dropdown actions | DropdownMenu |
| Search/select command | Command |
| Form input | Input |
| Text area | Textarea |
| Select | Select |
| Status | Badge |
| Grouped content | Card |
| Dense records | Table |
| Warning/info | Alert |
| Tabs | Tabs |
| Separator | Separator |
| Tooltip | Tooltip |
| Loading placeholder | Skeleton |

## Rules

- Do not invent component names.
- Do not hardcode colors when semantic tokens exist.
- Do not use inline styles unless prototype-only.
- Use accessible labels.
- Use existing Radius patterns first.
- For HTML prototype, simulate shadcn structure using clean class names.
