# Radius UX Patterns

## Web CRM Patterns

Use:
- dense table layouts
- filters near list views
- right-side detail panels
- concise row actions
- status badges
- inline validation
- clear empty states
- confirmation dialogs for destructive actions

Avoid:
- large banners unless explicitly requested
- unnecessary icons
- hidden primary actions
- pagination-heavy designs when simple list works better
- changing established patterns

## Transaction Patterns

Common nouns:
- transaction
- client
- offer
- listing
- envelope
- signing order
- CDA
- collaborator
- vendor
- auditor
- document

Every transaction flow should consider:
- assigned agent
- team visibility
- TC access
- status
- document state
- audit state
- next action

## Mobile Patterns

Use:
- bottom sheets
- grouped cards
- sticky CTAs
- readable status sections
- dark mode surfaces

## Copy Style

Use concise professional labels and real estate nouns.

Avoid:
- manage everything
- stuff
- item
- vague AI copy
